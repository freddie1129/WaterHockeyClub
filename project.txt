Project Topic: Underwater Hockey Club
Project Function Requirements:

1. Functionality planned for all users.
    a. Seeing the club news on the homepage
    b. Seeing the club's match schedule. The incoming three is shown on the homepage and all are shown on the match page.
    c. Seeing the team's introduction, and every team member's introduction including a photo and a paragraph.
    d. Seeing the club's introduction on the About page.
    e. Seeing the club's contact information on the Contact page.
    f. Become a general registered user by submitting the registration form. 

2. Functionality planned for registered users only.
    a. There are two kinds of registered user, one for the general registered user, other for the website's administrator.
    b. All anonymous user can become a general registered user by submitting the registration form with some valid inputs.
    c. The website's administrator permission can only be granted by the club's manager via the backend database interface. There is no interface on the webpage for anybody to upgrade to be a website's administrator.
    d. A general registered user can comment the club news and matches.
    e. The website's administrator has the permission to publish news, update the matches and score data, update the team member's information, etc.

3. Functionality planned for simple search.
    a. Search news by keywords.
    b. Search match by the team's name.
    c. Search match by match site. 
    
4. Functionality planned for the complex search.
    a. Search news by the combination of keywords and publish time.
    b. Search match by the combination of the team name, match site, team member, etc.
